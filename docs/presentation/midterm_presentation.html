Real Estate Platform

Midterm Presentation

Author: Eldar

Date: Nov 14, 2025

Why this topic?

  • Practical impact: buying/renting is a common, high‑friction process
  • Local market gaps: fragmented listings, inconsistent data, limited messaging
  • Good fit for full‑stack skills: UI/UX, APIs, auth, data modeling
  • Scalable: start simple, expand to maps, favorites, notifications

Existing solutions (overview)

  • Global platforms (e.g., Zillow): rich data, strong UX, limited relevance locally
  • Regional/local sites: basic filters, mixed data quality, weak user messaging
  • Social networks: reach but no structure (poor searchability and verification)

Opportunity: combine clean UX, structured filters, and integrated messaging tailored to local needs.

Goal by project end

  • End‑to‑end flow: discover → evaluate → contact → manage conversations
  • Owners/agents manage listings (create/edit/delete with images)
  • Buyers/renters search with meaningful filters (city, neighborhood, price, area, rooms, amenities)
  • Secure accounts and profile management (JWT, password change)
  • Documented REST API and prepared demo story

What’s implemented so far

  • Responsive React frontend (Vite), clean search and results
  • Property details with images and key features
  • User accounts: register, login, profile edit
  • Listing management: create, update, delete (owner‑only)
  • Messaging: send/receive with unread counts
  • TypeScript Express API + PostgreSQL (TypeORM)
  • Swagger docs at /api-docs

Architecture (high‑level)

  • Frontend: React SPA, modern UI and routing
  • Backend: Express (TypeScript), JWT auth, modular routes
  • Database: PostgreSQL via TypeORM (entities, migrations)
  • Docs: Swagger for quick testing and onboarding
  • Deployment: Docker files prepared

Plan & work dynamics (until semester end)

  • Week 1–2: UX polish, accessibility, image handling improvements
  • Week 3: Favorites & saved searches (with simple alerts)
  • Week 4: Admin/moderation basics, report/flag content
  • Week 5: Map view and location filters (Google Maps links)
  • Week 6: Performance, testing, documentation, demo rehearsal

Risks managed continuously; scope adaptable to time constraints.

Risks & mitigations

  • Data quality → validation, required fields, structured enums
  • Security → JWT, ownership checks, password rules
  • Images/storage → limit sizes, first image as cover, CDN later
  • Performance → server‑side filters, indexes, incremental profiling

Demo plan (midterm)

  1. Search by city/neighborhood and filter by rent/buy and price
  2. Open property details, show gallery and features
  3. Login and create a new listing with images
  4. From another account, send a message about that listing
  5. Show unread count and the conversation thread
  6. Open /api-docs to highlight API coverage
